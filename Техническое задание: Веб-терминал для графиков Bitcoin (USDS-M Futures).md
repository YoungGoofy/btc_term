# Техническое задание: Веб-терминал для графиков Bitcoin (USDS-M Futures)

## 1. Общее описание
Разработка высокопроизводительного браузерного веб-приложения (Single Page Application) для отображения графиков деривативов Binance (фьючерсов USDS-M) в реальном времени. Приложение должно строить свечи Heikin Ashi, отображать набор технических индикаторов в изолированных панелях и поддерживать мгновенное переключение таймфреймов (1m, 5m, 15m, 4h) через пользовательский интерфейс или горячие клавиши.

## 2. Стек технологий и среда разработки
* **Язык:** TypeScript.
* **Среда разработки и сборка:** Node.js, пакетный менеджер `npm` или `pnpm`. Для быстрой сборки рекомендуется использовать бандлер `Vite`.
* **Ядро UI (Фреймворк):** React 18+ (или Vue 3). Обеспечивает реактивное обновление состояния без прямого манипулирования DOM.
* **Стилизация (Layout):** Tailwind CSS для построения адаптивной CSS Grid/Flexbox сетки панелей.
* **Графика (Чартинг):** `lightweight-charts` (от TradingView). Библиотека на базе HTML5 Canvas, идеально подходящая для финансовых графиков. Она берет на себя автоматическое масштабирование осей, плавную отрисовку без мерцаний и поддержку десятков тысяч свечей.
* **Индикаторы:** Библиотека `technicalindicators` (npm) для расчета RSI, MACD и EMA. Логика для Heikin Ashi, VWAP и Pivots HL 10 реализуется разработчиком вручную.
* **Сеть:** Нативный браузерный `WebSocket` API для стримов и `fetch` API для начальной загрузки исторических данных.

## 3. Пользовательский интерфейс (UI)
Интерфейс представляет собой сетку (CSS Grid), занимающую `100vh` (всю высоту окна браузера) и разделенную на три фиксированные горизонтальные панели.

* **Панель 1 (Верхняя, ~50% высоты):**
  * График цены **BTCUSDT** (тип графика: `CandlestickSeries`, данные трансформируются в Heikin Ashi).
  * Линия индикатора **VWAP** (тип: `LineSeries`), наложенная поверх свечей на ту же ось Y.
* **Панель 2 (Средняя, ~25% высоты):**
  * Линии **EMA 9** и **EMA 21** (`LineSeries`).
  * Индикатор **Pivots HL 10** (`LineSeries` в виде ступенек или точечный график).
  * *Требование:* График должен иметь собственную, независимую от первой панели шкалу цен (Y-axis), которая автоматически масштабируется библиотекой `lightweight-charts` под текущие значения индикаторов.
* **Панель 3 (Нижняя, ~25% высоты):**
  * **MACD** (Гистограмма — `HistogramSeries`, линия MACD и линия Signal — `LineSeries`).
  * **RSI** (`LineSeries`).
  * *Требование:* Панель визуально разделяется на две колонки (50/50 по ширине). В левой части рендерится инстанс графика для MACD, в правой — для RSI. Для RSI добавляются статические фоновые линии уровней 30 и 70.

**Навигация:** Всплывающее модальное окно (Modal component) или фиксированный Header для выбора таймфрейма (1m, 5m, 15m, 4h). Окно должно вызываться по клику или горячей клавише (например, `Tab` или `F`).

## 4. Источник данных: Binance USDS-M Futures API
Интеграция состоит из двух этапов: загрузка исторического "среза" (REST) для расчетов и подхват живого потока (WebSocket). *Примечание: Binance API поддерживает CORS, поэтому запросы можно делать напрямую из браузера без промежуточного бэкенда.*

### 4.1. REST API (Исторические данные для "прогрева")
Для корректного расчета EMA и MACD на текущей свече требуется история.
* **Endpoint:** `GET https://fapi.binance.com/fapi/v1/klines`
* **Параметры:** `symbol=BTCUSDT`, `interval={таймфрейм}`, `limit=500`.

### 4.2. WebSocket API (Живой поток)
* **Base URL:** `wss://fstream.binance.com/ws`
* **Stream:** `<symbol>@kline_<interval>` (например, `btcusdt@kline_1m`). *Символ должен быть в нижнем регистре.*

**Интерфейсы TypeScript для десериализации:**
```typescript
// Основное сообщение вебсокета
interface BinanceWSMessage {
  e: string; // Тип события ("kline")
  E: number; // Время события
  k: BinanceWSKline; // Данные свечи
}

// Вложенный объект свечи
interface BinanceWSKline {
  t: number;  // Начало свечи
  T: number;  // Конец свечи
  o: string;  // Цена открытия
  h: string;  // Максимум
  l: string;  // Минимум
  c: string;  // Закрытие
  v: string;  // Объем торгов
  x: boolean; // Флаг закрытия свечи
}
```

## 5. Жизненный цикл и потоки данных (State Management)
Жизненный цикл управляется хуками React (например, `useEffect` для подписок) или стором состояния.

1. **Инициализация или смена таймфрейма:**
   * Очистка старых данных на графиках (вызов методов `.setData([])` у серий `lightweight-charts`).
   * Показ UI-индикатора загрузки (Spinner/Skeleton).
   * Отправка асинхронного `fetch` запроса на REST API.
   * Парсинг ответа, расчет Heikin Ashi и массивов индикаторов.
   * Передача массивов в графики (`series.setData(data)`).
2. **WebSocket Подписка:**
   * Открытие WS-соединения через `new WebSocket()`. Назначение обработчика на событие `onmessage`.
3. **Обновление состояния (Real-time Update):**
   * В обработчике `onmessage` парсится входящая свеча.
   * Производится инкрементальный расчет: обновление текущих значений Heikin Ashi, VWAP, EMA, MACD, RSI.
   * Если время новой свечи совпадает с последней — вызывается метод `series.update(newDataPoint)` для каждой линии и графика.
   * График плавно и без перерисовок всего холста отрисует изменения последнего бара/линии.
   * Если время больше — метод `series.update()` автоматически добавит новую свечу в конец графика.
