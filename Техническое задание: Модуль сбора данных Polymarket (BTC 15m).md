# Техническое задание: Модуль сбора данных Polymarket (BTC 15m)

## 1. Общая архитектура и API-домены
Для выполнения задачи агенту потребуется взаимодействовать с двумя из трех официальных API платформы:
* **Gamma API (`https://gamma-api.polymarket.com`)**: Отвечает за обнаружение событий и рынков. Используется для поиска маркетов «BTC 15m» и извлечения результатов (кто победил в прошлых 15-минутках).
* **CLOB API (`https://clob.polymarket.com`)**: Отвечает за финансовые данные. Через него запрашиваются снапшоты ордербука, спреды и устанавливается WebSocket-соединение для потоковой передачи.

## 2. Слой 1: Инициализация и Исторические данные (REST)
Эти методы агент должен использовать для получения контекста перед началом потокового мониторинга.

* **Поиск активного рынка и истории:**
    * *Эндпоинт:* Gamma API `GET /events` (или `GET /markets`).
    * *Логика:* Агент ищет событие по ключевым словам "BTC" и фильтрует 15-минутные интервалы. Отсюда извлекаются массив токенов (asset/token IDs для UP и DOWN). 
    * *История:* В этом же ответе (или при фильтрации по параметру `closed=true` / `resolved=true`) извлекается история предыдущих периодов и выигрышный исход.
* **Начальный снапшот ордербука и спреда:**
    * *Эндпоинт:* CLOB API `GET /book` (для всей глубины стакана) и `GET /spread` (для быстрого получения текущего спреда).
    * *Параметры:* Передается `token_id` нужного исхода (UP или DOWN).

## 3. Слой 2: Данные реального времени (WebSocket)
Для отслеживания цен и объемов без задержек и без риска упереться в Rate Limits (лимиты на количество REST-запросов), модуль должен использовать официальный **Market Channel**.

* **Подключение:** `WSS` канал к серверам CLOB API.
* **Подписка (Subscription Request):** Агент отправляет JSON-запрос вида `{"type": "market", "assets_ids": ["ID_UP", "ID_DOWN"]}`.
* **Обрабатываемые события (Events):**
    * `price_change`: Возвращает актуальные `best_bid` и `best_ask`. На основе этих двух значений модуль локально вычисляет размер спреда и обновляет текущие цены UP/DOWN в памяти для агента.
    * `last_trade_price`: Срабатывает при каждой реальной сделке. Содержит параметры `size` (объем) и `price`. Суммируя `size`, модуль рассчитывает актуальный проторгованный объем за 15-минутное окно.

## 4. Спецификация Tool-ов для агента
В интерфейс фреймворка (например, OpenClaw) нужно экспортировать три основных инструмента:

1.  **`fetch_btc_market_context()`**
    * Делает REST-запросы к Gamma API. Возвращает ID текущих токенов и результаты последних 5 завершенных 15-минуток.
2.  **`start_market_stream(assets_ids)`**
    * Запускает фоновый асинхронный процесс подключения к WebSocket `Market Channel`.
3.  **`get_live_metrics()`**
    * Синхронно возвращает агенту последние рассчитанные данные из фонового WSS-потока (Цены UP/DOWN, Спред, Накопленный объем).

## 5. Сетевая инфраструктура и окружение
Поскольку Polymarket блокирует доступ к API и депозитам из определенных юрисдикций, сетевой слой должен быть готов к маршрутизации трафика. Если для доступа к сайту из браузера уже используется настроенный прокси-туннель (например, через профиль ZeroOmega), то URL этого же SOCKS5/HTTP сервера необходимо явно прокинуть в переменные окружения агента. Библиотеки, используемые для сети (например, `aiohttp` для REST и клиент `websockets`), должны пропускать все запросы к Gamma и CLOB API через этот шлюз.

Сборку окружения для самого скрипта-коннектора (установка нужных версий Python, `aiohttp` и зависимостей агента) наиболее надежно будет изолировать через Devbox, чтобы избежать конфликтов глобальных пакетов при работе с сетевыми потоками.
